Java从头再来-垃圾收集

发表于 2018-04-04 | 分类于 Java | | 阅读次数:

都说Java的垃圾我们不需要管，但是垃圾收集之类的相关知识我们还是有必要了解一下的，做到知己知彼百战不殆。

回收算法简介

  • 标记清理算法

    这个算法相对而言是最近简单的，标记阶段标识出哪些内存可以被清理，在清理阶段将这些内存清理掉。算法的优点显而易见–快速高效，但是缺点也比较明显，容易产生内存碎片，导致后续分配大对象空间的时候没有连续空间，进而引发Full GC。

  • 复制算法

    把内存一份为二，每次用其中一块内存，每次内存清理的时候，将那块内存上还存活的对象复制到另外一块内存上，这样就不会出现内存碎片了，但是内存的利用率太低了。

  • 标记整理算法

    标记整理算法算是标记清理算法和复制算法的结合体，在标记清理完垃圾内存后，会对存留下来的对象做一次整理，这样就能避免产生内存碎片的问题。

结构分析

目前广泛采用的Hotspot JVM主要把新生代内存分成三部分：一块Eden区和两块Survivor区（From Survivor, To Survivor），比例为8:1:1，一般情况下，新的对象都会被分配到Eden区，如果这个时候Eden区满了，就会发生一次Minor GC，还存活的对象将会被转移到Survivor区内，同一时间只有一个Survivor区会和Eden区配合使用，每次Minor GC之后，Survivor区内的对象如果没有被回收掉，它的年龄都会加一，直到一定阈值之后，它会被放到老年代中（默认16）
