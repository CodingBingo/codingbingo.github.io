Java从头再来-并发

发表于 2018-04-13 | 分类于 Java | | 阅读次数:

在深入Java并发相关只是之前，我们先来了解一下必要的相关知识。

Tips

  • CAS
    CAS全程是compareAndSet，这个主要使用的Unsafe这个类，这个类提供了一下硬件级别的原子操作，而compareAndSet有三个参数（内存值、预期值、新值），操作的时候，如果内存值与预期值相等，则更新内存值为新值，否则啥都不做。
  • synchronized
    synchronized 作用于静态方法时是对 Class 对象加锁，作用于实例方法时是对实例加锁。

LockSupport.class简介

这个类是Java中用来创建程序阻塞和唤醒的元语句。一般分别采用LockSupport.park()和LockSupport.unpark()来创建线程的阻塞和唤醒。

并发讲解

想掌握好Java的并发，那么java.util.concurrent包是你必须要好好了解的，这里先看一下这个package里面到底都有些啥吧。

  • Atomic数据类型
    这里定义了我们常见的如：Interge、Long、Boolean等原子化操作的实现。这些原子化的操作也都是通过CAS去实现的，有兴趣可以看下get和set方法的实现。
  • 锁
    定义了锁操作相关的一些类，包括我们常见的：可重入锁（公平锁、非公平锁…）、读写锁等等。
  • 集合类并发实现
    实现了如List、Queue、Map等相关集合类的并发实现。
  • 多线程
    这个主要就是多线程任务并发执行时，对任务的管理等相关操作。

Lock

关于锁的操作，应该是Java并发这一块，最重要的一部分内容了，下面我们来看一下Lock这个类具体都有啥。

这里我们以可重入锁ReentrantLock这个类为主线看一下，具体都是怎么实现的。

总的来说，ReentrantLock是属于一种独享、可重入，支持公平非公平的一种锁结构。其中重点还是在锁的获取、释放之类的操作，这部分主要是涉及到BlockingQueue，我们放到下面一部分。

BlockingQueue

其实简单点说，BlockingQueue就是一个先进先出(FIFO)的队列(Queue)，不过这个队列比较特殊，因为它会阻塞。当队列为空的时候，有线程过来取东西，这个时候会被阻塞等到队列里面有元素；当队列满了的时候，这个时候添加元素会被阻塞直到队列里可以放入新元素再放入队列中。
