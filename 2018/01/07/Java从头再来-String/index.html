String

String本质上还是由字符数组构成的，类本身实现了三个接口Serializable、Comparable、CharSequence。

  • Serializable
    这里简单说一下Serializabl这个接口，本身其实没有任何属性和方法，所以如果某个类需要序列化（序列化在Java中主要是保证在各个平台的一致性。），只需要实现这个接口接口。
  • Comparable
    Comparable接口主要是要实现compareTo这个方法。相信大家对于compareTo这个方法一定不会陌生。
  • CharSequence
    CharSequence主要提供了一系列针对字符数组操作的方法。

String、StringBuffer、StringBuilder

这里看一下我们经常一起比较的三个字符串相关的类。String类是一个不可变类，也就是说如果有一个字符串是String类型的，那么如果想改变这个字符串的长度是不行的。StringBuffer和StringBuilder都是可变的，它们都是继承自AbstractStringBuilder，但是有个比较大的不同点事，StringBuffer几乎所有的方法都是synchronized的，也就是说StringBuffer是线程安全的，但是StringBuilder是线程不安全。我们都知道synchronize是要以性能为代价来保证线程安全的，所以说如果是单线程状态下，还是推荐使用String或者StringBuilder的。另外，一般情况下，StringBuilder和StringBuffer每次需要扩容的时候都会扩容到原来长度的两倍加二（即：oldSize * 2 + 2,这个是经验值哈，空对象的初始大小都是16）
